# MirrorBreak Protocol

**Empirical Pattern Deconstruction Model (EPDM)**

A systematic framework for understanding human behavioral patterns through empirical observation and structural analysis.

---

## Core Philosophy

> **Observe → Hypothesize → Stress-test → Affirm/Reject → Refine → Structural Extract**

MirrorBreak is not about labeling personalities. It's about understanding the adaptive systems that drive human behavior.

---

## Key Principles

1. **Humans are Adaptive Systems** — Not fixed types. What we observe are: adaptations, compensations, defense structures, energy directions.

2. **Every Pattern is an Adaptation** — There's no "bad" trait. Only adaptations to specific pressures.
   - Excessive control → adaptation to uncertainty
   - People pleasing → adaptation to relational threat

3. **Contradiction is More Informative than Consistency** — The system weights higher on: energy shifts, value tensions, narrative inconsistencies. That's where the core structure reveals itself.

---

## The Workflow

### Step 1: Observational Intake
Raw data collection:
- Answer content
- Sentence structure
- Information density
- Emotional tone
- Cognitive complexity
- Response to ambiguous questions

### Step 2: Multi-Hypothesis Generator
Minimum 3 competing hypotheses:
- H1 — Control-based adaptation
- H2 — Validation-based adaptation
- H3 — Security-based adaptation

All considered possible until tested.

### Step 3: Stress Testing
Covert questions targeting:
- Authority response
- Loss simulation
- Social rejection scenario
- Ambiguity tolerance
- Moral dilemma

The system doesn't seek confirmation. It seeks cracks.

### Step 4: Affirm / Reject
Each hypothesis scored on:
- Evidence support
- Contradiction weight
- Emotional shift magnitude

Weak hypotheses eliminated. Strong ones advance.

### Step 5: Structural Extraction
From strongest hypothesis, extract 5 core structures:
1. Core Fear
2. Core Drive
3. Primary Defense
4. Adaptive Strength
5. Relational Risk

Not labels. Dynamic structures.

## The Four Dimensions

### 1. Cognitive Structure Dimension
- Depth of causality
- System integration ability
- Binary vs gradient cognition
- Abstraction bandwidth
- Meta-cognitive awareness

### 2. Emotional Architecture Dimension
- Emotional granularity
- Regulation strategy
- Shame vs guilt bias
- Vulnerability threshold
- Recovery speed

### 3. Power & Authority Dimension
- Dominance expression
- Compliance adaptation
- Control orientation
- Recognition sensitivity
- Social positioning strategy

### 4. Adaptive Core Dimension
Outputs:
- Origin pressure (formative environment)
- Defense pattern
- Compensation style
- Latent capacity

- Testable

**This isn't mysticism. This is iterative structural modeling.**
